nginx工作得几个重要阶段

发表于 2020-09-13 分类于 服务器

nginx常见变量

名称 作用 限制
$proxy_add_x_forwarded_for 如果$http_x_forwarded_for不为空,等同于’$http_x_forwarded_for,$remote_addr’，否则等于$remote_addr
$remote_addr 上一个http请求的ip 有可能被realip模块修改
$realip_remote_addr 上一个http请求的ip realip模块提供的变量
$http_x_forwarded_for http request header中的X-Forwarded-For的值

postread阶段

对应 http-real-ip模块
|指令或变量| 格式|说明|
|—-|—-|—-|
|set_real_ip_from|address;CIDR;unix|定义受信地址。如CDN,内网..事先已知的反向代理。通过受信渠道的地址才会被处理，替换|
|real_ip_header|field;X-Real-IP;X-Forwarded-For;proxy_protocol|默认值X-Real-IP,指定用于替换$remote_addr的内容,常使用X-Forwarded-For|
|real_ip_recursive |可选 on;off|默认为off;off:$remote_addr将替换为real_ip_header的最后一个地址；on:$remote_addr将替换为real_ip_header最后一个非置信的ip|
|$realip_remote_addr||原始客户端地址|
|$realip_remote_port||原始客户端端口|

rewrite阶段

指令 格式 说明
return 返回值格式如 code text，或者code url 或者单独的url code 200,301等；url或text中可以包含变量
rewrite regex replacement flag

  • return

    • 返回值为单独的url时，相当于返回 302 url.
    • As a special case, a redirect URL can be specified as a URI local to this server, in which case the full redirect URL is formed according to the request scheme ($scheme) and the server_name_in_redirect and port_in_redirect directives.

  • rewrite

    • rewrite flag值可取last,break,redirect,permanent.
    • last 新的uri再次进行location匹配，
    • break ,完成当前在location中剩余操作
    • redirect 返回302暂时重定向；如果replacement以http(s)开头也会触发
    • permanent 永久重定向 返回301
      两个重定向区别：永久重定向，相当于告诉客户端下次请求直接转发到重定向；暂时重定向每次都重复这个过程。

findconfig阶段

此阶段主要是对location的匹配。

  • 对请求的url序列化。例如，对%xx等字符进行解码，去除url中多个相连的/，解析url中的.，..等。这一步是匹配的前置工作。

  • location uri有两种表示形式,一种是使用普通字符串,一种使用正则.

  • 对于普通字符串的匹配规律。首先将所有普通字符串按字典序进行排序。请求uri逐一扫描，当当前字符串大于请求uri时停止，并选取最后一个匹配。（精确匹配除外，且所有普通字符串不能重复。）

  • 匹配顺序

    1. 首先进行精确匹配，如果匹配到=修饰的location，停止匹配。
    2. 然后检查普通字符串定义的其余location，记录最长匹配的location字符串。如果记录的最长匹配字符串用^~修饰,则停止匹配。
    3. 接着按配置文件定义的顺序，进行正则location匹配.使用第一个匹配结果。
    4. 如果没有正则匹配,使用前面记录的最长前缀字符串的location.

  • location @name 主要用于内部重定向.注意命名location中不能再嵌套其它的命名location。

  • location中的字符有没有/都没有影响。也就是说/user/和/user是一样的.可以使用精确匹配测试.

  • 如果请求URL结构是https://domain.com/的形式，尾部有没有/都不会造成重定向。因为浏览器在发起请求的时候，默认加上了/.

  • 如果请求URL的结构是https://domain.com/some-dir/。尾部如果缺少/将导致重定向。因为根据约定，URL尾部的/表示目录，没有/表示文件。所以访问/some-dir/时，服务器会自动去该目录下找对应的默认文件。如果访问/some-dir的话，服务器会先去找some-dir文件，找不到的话会将some-dir当成目录，重定向到/some-dir/，去该目录下找默认文件。

    修饰符 含义 其它
    ~ 波浪线表示执行一个正则匹配，区分大小写
    ~* 表示执行一个正则匹配，不区分大小写 正则匹配顺序很重要,按顺序匹配，匹配到则停止
    ^~ ^~表示普通字符匹配，如果该选项匹配，只匹配该选项，不匹配别的选项，一般用来匹配目录
    = 进行普通字符精确匹配 使用=精准匹配可以加快查找的顺序，如果根域名经常被访问的话建议使用=。

preaccess阶段

限制并发连接数
|指令|格式|默认值|说明|
|–|–|–|–|
|limit_conn_zone|key zone=name:size rate=rate||在http，用于定义共享内存,处理速度rate单位为r/s或者r/m|
|limt_conn|zone number||限制并发连接数,用于http,server,location|
|limit_conn_status|code|503|限制发生时向客户端返回的错误码，用于http,server,locacion|
|limit_conn_log_level|info;notice;warn;eror|error|限制发生时日志级别，用于http,server,locacion|
|limit_req_zone|key zone=name [burst=number] [nodelay]|burst=0,nodela立刻处理||
|limit_req_status|code|503|限制发生时向客户端返回的错误码，用于http,server,locacion|
|limit_req_log_level|info;notice;warn;eror|error|限制发生时日志级别，用于http,server,locacion|

access阶段

对ip进行限制的阶段，通过顺序为access->auth_basic->auth_request
|指令|格式|默认值|说明|
|–|–|–|–|
|allow| address;CIDR;unix|all||用于http,server,location,limit_except中|
|deny| address;CIDR;unix|all||同allow，多条指令按顺序执行，直到匹配成功，用于控制访问ip|
|auth_basic|string;off|off，取消从上级模块继承的结果；字符串指定一个域。|用于http,server,location,limit_except中|
|auth_basic_user_file|file||同auth_basic属于auth_basic模块的指令，指定验证用户的文件|
|auth_request|uri;off|off|用于http,server,location中，向上游服务转发请求，如返回2xx，则继续执行；若401或403则被拒绝并将响应返回客户端；其它码视为一个错误|
|auth_request_set|$variable value||同auth_request,用于设置变量，这两个变量属于auth_request模块，默认未编译进nginx|
|satisfy|all;any|all|对access阶段的各个模块结果取并或者或结果|

precontent阶段

  • try_files模块
    用于location中，根据请求uri进行逐个文件的匹配，返回第一个匹配项。末尾必须跟着一个fallback,它是一个命名location，或者存在的uri,如 =404状态码。

    content阶段

  • root和alias

    • root: 用于http, server, location, if in location块中.默认值 html.指定请求uri的前缀.root加作为前缀加到uri上.root与rewrite搭配实现alias的效果.
    • alias:用于location中,alias将替换 location匹配内容,形成新的uri.对于正则location,应包含捕获从而在alias中可以引用

  • index
    指定目录时返回默认的文件，默认配置为index index.html;可以在serve,location,http中配置。** 注意**index会引起内部重定向，导致文件在不同的location中处理.
